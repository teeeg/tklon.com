## What is this

Personal blog at tklon.com. The site is generated by a single Rust binary
(`tklon`); the content is Markdown + committed media. Pushing to master
triggers a GitHub Actions workflow that builds the site and syncs it to
S3/CloudFront.

## Directory structure

```
tklon.com
├── .github/workflows/deploy.yml --> builds + deploys on push to master
├── Cargo.toml                   --> workspace (so `cargo run` works from root)
├── Makefile                     --> AWS layer: infra / publish / deploy / stats
├── deploy
│   ├── lambda/                  --> stats report Lambda
│   ├── params.yml               --> per-deployment values
│   └── template.yml             --> CloudFormation template
├── generator                    --> the tklon SSG (Rust); builds the `tklon` binary
│   ├── src/                     --> generator source
│   └── assets/tags.js           --> client-side tag filter
└── site                         --> site content
    ├── config.yml               --> site identity (name, urls, author, bucket)
    ├── templates/               --> Tera page templates
    ├── source/posts/{year}/     --> Markdown posts
    ├── source/pages/            --> standalone pages (about, …)
    ├── source/stylesheets/      --> SCSS (compiled in-binary via grass)
    ├── source/images/           --> committed responsive variants (avif/webp)
    ├── source/fonts/
    ├── data/                    --> images.json / videos.json manifests
    └── images/                  --> image masters (jpg)
```

## Local development

Requirements: [Rust](https://www.rust-lang.org/) (cargo). A workspace manifest at
the repo root lets cargo run from anywhere in the tree.

```sh
cargo run -- serve      # dev server at http://localhost:4567, rebuilds on save
cargo run -- build      # production build into build/
```

Or install the command once and call it directly:

```sh
cargo install --path generator
tklon serve
```

Authoring a post: drop `site/source/posts/{year}/{slug}.md` with `title` / `date`
/ `tags` front matter. Images use native Markdown against a manifest name —
`![alt](image-name "caption")` — and expand to a responsive `<picture>`. Video
and embeds use `{{< video name="…" caption="…" >}}` / `{{< embed url="…"
title="…" >}}`. See [`generator/README.md`](generator/README.md) for details.

## Posting from your phone

A photo post needs no laptop — the whole loop is in the GitHub mobile app:

1. **Actions → "new post" → title → Run** — creates a dated branch, a stub with
   today's date filled in, and a draft PR (`.github/workflows/new-post.yml`).
2. **Edit the post** in the PR: write prose, and drop `![alt text](@)` wherever a
   photo goes — one `@` per photo, in order.
3. **Attach the photos** to the PR description, in that same order.
4. `process-media.yml` downloads them, fills the `@` slots (`tklon ingest`),
   encodes variants + backs up masters (`tklon images`), commits the result, and
   keeps one **sticky preview comment** (thumbnails + names) in sync per commit.
5. Green checks → **merge → live.**

Notes:
- Photos via attachments only (GitHub caps free-plan attachments at ~10 MB). For
  video, encode on desktop with `tklon video`, or use GitHub Pro (100 MB).
- HEIC isn't supported — iOS normally converts to JPEG on upload; otherwise the
  run fails loudly rather than shipping a broken image.
- Needs the `tklondotcom-media-backup` IAM role (in `deploy/template.yml`) — run
  `make infra` once to create it.

## Deploying

Content and infrastructure deploy separately:

- **Content** — pushing to master runs `.github/workflows/deploy.yml`, which
  builds the site with `cargo run --release -- build` and syncs it to S3 +
  invalidates CloudFront. The workflow authenticates to AWS via OIDC (no stored
  keys).
- **Infrastructure** in `template.yml` is applied manually with `make infra`.

With the `aws-cli` installed and authenticated (e.g. `aws login`):
`make infra` to deploy/update the CloudFormation stack;
`make publish` to build + sync content locally (manual deploy, bypasses Actions);
`make help` to list every target.

Stack name and region default to `tklondotcom` / `us-east-1`; override inline,
e.g. `make infra STACK=foo REGION=us-west-2`.

Per-deployment values live in [`deploy/params.yml`](deploy/params.yml) (single
source of truth). `make infra` reads it and passes values to CloudFormation;
`template.yml` is the schema only and is not standalone-deployable without
`--parameter-overrides`.

- **AcmCertificateArn** ACM certificate [ARN](https://docs.aws.amazon.com/general/latest/gr/aws-arns-and-namespaces.html) covering the apex + www subdomain (must live in us-east-1)
- **Route53HostZoneName** Route 53 hosted zone where alias records to CloudFront are created
- **GitHubRepositoryOwner** GitHub account or org that owns the repo (scopes the OIDC deploy role)
- **GitHubRepositoryName** GitHub repository name (scopes the OIDC deploy role)
- **StatsReportEmail** address that receives the weekly access-log summary; SNS emails a confirmation link on first create/update — click it to start delivery

## Media authoring

- **Images** — drop a master in `site/images/` (gitignored; `tklon images` backs
  it up to S3), run `tklon images` to (re)generate the committed AVIF/WebP variants
  + `site/data/images.json`, then reference it in a post as `![alt](master-name "caption")`.
- **Videos** — drop a clip in `site/videos/` (gitignored), run `tklon video clip.mov`
  to encode + upload the MP4 to S3, back up the master, extract a poster, and
  record it in the committed `site/data/videos.json`; reference it with
  `{{< video name="clip" caption="…" >}}`. `tklon video --check` (suitable as a
  pre-push hook) keeps sources and manifest in sync. Requires ffmpeg + aws.

See [`generator/README.md`](generator/README.md) for the full behaviour.
